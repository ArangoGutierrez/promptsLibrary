## ACTUAL VIABLE DISPATCH IDIOM: direct httpx (OpenAI-compatible)
NAT's NIM endpoint is OpenAI-compatible. The viable _invoke_nat() idiom is:

```python
import httpx, os

def _invoke_nat(panelist, messages, api_key):
    """Direct httpx call to NIM/OpenAI-compatible endpoint."""
    endpoint = os.environ.get("CLAUDE_PANEL_DA_ENDPOINT", "")
    https_proxy = os.environ.get("HTTPS_PROXY", "")
    payload = {
        "model": panelist.model,
        "messages": messages,
        "max_tokens": panelist.max_tokens,
        "temperature": panelist.temperature,
    }
    with httpx.Client(proxy=https_proxy or None, timeout=panelist.timeout_seconds) as client:
        resp = client.post(
            endpoint,
            headers={"Authorization": f"Bearer {api_key}", "Content-Type": "application/json"},
            json=payload,
        )
        resp.raise_for_status()
        data = resp.json()
        content = data["choices"][0]["message"]["content"]
        return content  # str
```

This is 12 LOC. httpx 0.28.1 is already installed. NIMModelConfig fields are only
used for config validation, not for the actual HTTP call path.

## Response shape (live call confirmed)
endpoint: https://inference-api.nvidia.com/v1/chat/completions
model: nvidia/nvidia/nemotron-3-super-v3 (note: env has double-prefix, use as-is)
response type: dict (JSON parsed)
text attribute: data["choices"][0]["message"]["content"]  → str
finish_reason: "stop" when complete; "length" when truncated (content=None when truncated)
reasoning model: has "reasoning_content" field alongside "content"
IMPORTANT: content is None when finish_reason="length" (truncated). dispatch-da.sh already
handles this case with write_error. dispatch.py must do the same.

## Anthropic / OpenAI parallels
No nat.llm.anthropic_llm module in 1.6.0.

For "nat-anthropic" backend: use httpx + Anthropic Messages API directly (different JSON schema).
For "nat-openai" backend: same httpx idiom as NIM (OpenAI-compatible).

## Notes
1. NAT Builder dispatch is BLOCKED without framework packages. Do NOT attempt WorkflowBuilder
   path in dispatch.py. Use direct httpx instead.
2. NIMModelConfig is only useful for field validation / config parsing. The actual dispatch
   bypasses NAT's Builder entirely.
3. ALL_PROXY=socks5h://localhost:60546 causes httpx ImportError (needs socksio).
   Use HTTPS_PROXY (http://localhost:60541) explicitly in httpx.Client(proxy=...).
4. NIMModelConfig accepts both model_name= and model= (validation_alias). Panelist.model
   maps to model_name.
5. max_tokens must be sufficient for reasoning models: with 50 tokens content=None; 512 tokens OK.
   dispatch.py should use panelist.max_tokens (default 32768 from config.py).
6. CLAUDE_PANEL_DA_ENDPOINT env var is shared between dispatch-da.sh and dispatch.py.
   The nat-nim backend is the same NIM endpoint the DA shell script already uses.
